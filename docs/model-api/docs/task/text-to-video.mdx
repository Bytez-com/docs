<AccordionGroup>
  <Accordion defaultOpen title="Basic usage">
    Send a text prompt to generate a video output (usually in mp4):

    <CodeGroup>
      ```javascript javascript
      import Bytez from "bytez.js";

      const sdk = new Bytez("BYTEZ_KEY");
      const model = sdk.model("ali-vilab/text-to-video-ms-1.7b");

      const { error, output } = await model.run("A cat playing with a rose");

      console.log({ error, output });
      ```
      ```python python
      from bytez import Bytez

      sdk = Bytez("BYTEZ_KEY")
      model = sdk.model("ali-vilab/text-to-video-ms-1.7b")

      output, error = model.run("A cat playing with a rose")

      print({ "error": error, "output": output })
      ```
      ```bash http
      curl -X POST 'https://api.bytez.com/models/v2/ali-vilab/text-to-video-ms-1.7b' \
      -H 'Authorization: Key BYTEZ_KEY' \
      -H 'Content-Type: application/json' \
      --data '{
        "text": "A cat playing with a rose"
      }'
      ```
    </CodeGroup>

  </Accordion>
    <Accordion title="Stream">
    Instead receiving an video file in JSON format, you opt to stream back the data URL:

    <CodeGroup>
      ```javascript javascript
      import Bytez from "bytez.js";

      const sdk = new Bytez("BYTEZ_KEY");
      const model = sdk.model("ali-vilab/text-to-video-ms-1.7b");
      const text = "A cat playing with a rose"
      const stream = true

      const readStream = await model.run(text, stream);

      // the stream returns the base64 data url
      for await (const chunk of readStream) {
        console.log(chunk)
      }
      ```
      ```python python
      from bytez import Bytez

      sdk = Bytez("BYTEZ_KEY")
      model = sdk.model("ali-vilab/text-to-video-ms-1.7b")
      ```
